# 国际化 (i18n) 指南

## 概述

Paperclip 使用 `i18next` 和 `react-i18next` 实现国际化支持。当前支持的语言：

- 🇺🇸 英语 (en) - 默认语言
- 🇨🇳 中文 (zh)

## 快速开始

### 在组件中使用翻译

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <div>
      <h1>{t("nav.dashboard")}</h1>
      <button>{t("common.save")}</button>
    </div>
  );
}
```

### 使用 useLanguage Hook

```tsx
import { useLanguage } from "@/hooks/useLanguage";

function LanguageSelector() {
  const { language, changeLanguage, t } = useLanguage();
  
  return (
    <select 
      value={language} 
      onChange={(e) => changeLanguage(e.target.value)}
    >
      <option value="en">English</option>
      <option value="zh">中文</option>
    </select>
  );
}
```

### 使用 LanguageSwitcher 组件

```tsx
import { LanguageSwitcher } from "@/components/LanguageSwitcher";

function Header() {
  return (
    <header>
      <LanguageSwitcher />
    </header>
  );
}
```

## 添加新的翻译

### 1. 在翻译文件中添加键值对

编辑 `ui/src/i18n/locales/en/translation.json` 和 `ui/src/i18n/locales/zh/translation.json`：

```json
{
  "myFeature": {
    "title": "My Feature Title",
    "description": "Description of my feature"
  }
}
```

### 2. 在中文翻译文件中添加对应的翻译

```json
{
  "myFeature": {
    "title": "我的功能标题",
    "description": "我的功能描述"
  }
}
```

### 3. 在组件中使用

```tsx
const { t } = useTranslation();
<h2>{t("myFeature.title")}</h2>
<p>{t("myFeature.description")}</p>
```

## 翻译文件结构

翻译文件按功能模块组织：

```
ui/src/i18n/
├── config.ts              # i18n 配置
├── index.ts               # 导出
└── locales/
    ├── en/
    │   └── translation.json  # 英文翻译
    └── zh/
        └── translation.json  # 中文翻译
```

### 翻译键命名规范

- 使用点号分隔的命名方式：`模块。功能。具体项`
- 通用翻译放在 `common` 下
- 导航相关放在 `nav` 下
- 错误信息放在 `errors` 下
- Toast 提示放在 `toast` 下

## 动态插值

i18n 支持动态插值：

```json
{
  "time": {
    "minutesAgo": "{{count}} 分钟前"
  }
}
```

```tsx
t("time.minutesAgo", { count: 5 }) // 输出："5 分钟前"
```

## 语言检测

语言检测按以下顺序进行：

1. localStorage 中保存的语言设置
2. 浏览器语言设置
3. 默认语言（英语）

## 添加新语言

### 1. 创建新的翻译文件

```
ui/src/i18n/locales/ja/translation.json  # 日文
```

### 2. 更新配置文件

在 `ui/src/i18n/config.ts` 中添加新语言：

```ts
import jaTranslation from "./locales/ja/translation.json";

export const resources = {
  en: { translation: enTranslation },
  zh: { translation: zhTranslation },
  ja: { translation: jaTranslation },
} as const;
```

### 3. 更新 LanguageSwitcher 组件

在 `ui/src/components/LanguageSwitcher.tsx` 中添加新语言选项：

```ts
const languages = [
  { code: "en", label: "English", flag: "🇺🇸" },
  { code: "zh", label: "中文", flag: "🇨🇳" },
  { code: "ja", label: "日本語", flag: "🇯🇵" },
] as const;
```

## 最佳实践

1. **始终使用翻译键** - 不要在组件中硬编码文本
2. **保持翻译文件同步** - 添加新翻译时确保所有语言都有对应翻译
3. **使用有意义的键名** - 键名应该描述内容的用途，而不是内容本身
4. **避免嵌套过深** - 保持翻译键简洁
5. **测试所有语言** - 确保翻译在所有支持的语言下都能正常工作

## 注意事项

- 翻译文件应该是有效的 JSON 格式
- 不要翻译技术术语、品牌名称和代码
- 中文翻译应该使用简体中文
- 保持翻译的上下文一致性
